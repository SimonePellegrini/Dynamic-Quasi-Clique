# Dynamic Graph Experiments: NBSim & Credits Algorithm

This repository contains a C++ experimental framework designed to analyze and compare the performance of various graph algorithms — specifically `FastNBSim`, `DynamicNBSim`, and a custom Credits-based Algorithm — applied to **dynamic graphs**. The framework processes streams of edge insertions and deletions to evaluate how quasi-clique sizes, densities, and computational times evolve over time.

---

## 📁 Directory Structure

The executable strictly expects the following relative directory structure to locate datasets, load sequences, and save experimental results. **Ensure these folders exist before running the code.**

```
├── Datasets/
│   └── <fileName>.txt                   # Initial static input graph
│
├── Esperimenti/
│   └── <fileName>/                      # Root output directory for a specific dataset
│       ├── standard/                    # Outputs for standard operational sequences
│       ├── mixed/                       # Outputs for mixed operational sequences
│       ├── speedup.txt                  # (Auto-generated by baseline comparison)
│       └── tempoNBSim.txt               # (Auto-generated by baseline comparison)
│
└── Progetto/                            # Folder containing the source code
    ├── Sequences/
    │   ├── StandardSequences/           # e.g., <fileName>_<prob>.txt
    │   └── MixedSequences/              # e.g., <fileName>_<prob>_<type>.txt
    ├── main.c++                         # Main experimental driver
    ├── DynamicNBSim.c++
    ├── FastNBSim.c++
    ├── CreditsAlgorithm.cpp
    └── ...                              # Other headers/source files
```

## ⚙️ Requirements & Compilation

**Requirements:** A C++ compiler with C++11 support or higher, to utilize libraries such as `<chrono>`, `<random>`, and `<unordered_map>`.

**Compilation:** It is highly recommended to compile with the `-O3` optimization flag, as the code measures exact execution times for benchmarking.

```bash
g++ -O3 main.c++ -o run_experiments
```

---

## 💻 Usage

The program is executed via the command line and requires exactly **10 arguments** (in addition to the executable name).

**Syntax:**

```bash
./run_experiments <phi> <alpha> <k> <fileName> <p> <gamma> <b> <run_dynamic_baseline> <strategy> <type>
```

### Parameters

| # | Parameter | Type | Description |
|---|-----------|------|-------------|
| 1 | `<phi>` | float | ϕ parameter for the credits algorithm |
| 2 | `<alpha>` | float | α parameter for the credits algorithm |
| 3 | `<k>` | int | Target size/threshold parameter for the quasi-cliques |
| 4 | `<fileName>` | string | Name of the dataset (e.g., `youtube`, without the `.txt` extension) |
| 5 | `<p>` | float | Probability parameter used to fetch the correct sequence file. If `p = 0`, the system will run Insert-Only experiments |
| 6 | `<gamma>` | float | γ density parameter |
| 7 | `<b>` | float | Specific boundary/tuning parameter for the NBSim structure |
| 8 | `<run_dynamic_baseline>` | int | Set to `1` to run dynamic baseline experiments, or `0` to skip them |
| 9 | `<strategy>` | string | The sequence strategy to use (e.g., `standard`, `mixed`) |
| 10 | `<type>` | string | The graph generation or sequence type (e.g., `erdos`, `power`) |

**Example:**

```bash
./run_experiments 0.5 0.8 10 my_graph 0.1 0.9 0.6 1 standard erdos
```

## 📊 Output Artifacts

All results are automatically serialized to `.csv` and `.txt` files in the `../Esperimenti/<fileName>/` directory tree.

| File | Description |
|------|-------------|
| `times_dynamic_*.csv` | Execution time logs for the dynamic algorithm |
| `times_credits_*.csv` | Execution time logs for the credits algorithm |
| `results_dynamic_*.csv` | Structural results for the dynamic algorithm |
| `density_credits_*.csv` | Density metrics for the credits algorithm |
| `Qs_Size_b=*.csv` | Quasi-clique size metrics |
| `number_of_credits_*.csv` | Internal credit counts |
| `gamma_degree_*.csv` | Gamma-degree metrics |

> **Note:** All datasets must be preprocess using `generateSequences.py` before running experiments.
