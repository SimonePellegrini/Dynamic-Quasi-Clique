# Dynamic Graph Experiments: NBSim & Credits Algorithm

This repository contains a C++ experimental framework designed to analyze and compare the performance of various graph algorithms—specifically `FastNBSim`, `DynamicNBSim`, and a custom Credits-based Algorithm—applied to **dynamic graphs**. The framework processes streams of edge insertions and deletions to evaluate how quasi-clique sizes, densities, and computational times evolve over time.

## 📁 Directory Structure

The executable strictly expects the following relative directory structure to locate datasets, load sequences, and save experimental results. **Ensure these folders exist before running the code:**

```
text
├── Datasets/
│   └── <fileName>.txt               # Initial static input graph
├── Esperimenti/
│   └── <fileName>/                  # Root output directory for a specific dataset
│       ├── standard/                # Outputs for standard operational sequences
│       ├── mixed/                   # Outputs for mixed operational sequences
│       ├── speedup.txt              # (Auto-generated by baseline comparison)
│       └── tempoNBSim.txt           # (Auto-generated by baseline comparison)
└── Progetto/                        # Folder containing the source code
    ├── Sequences/
    │   ├── StandardSequences/       # e.g., <fileName>_<prob>.txt
    │   └── MixedSequences/          # e.g., <fileName>_<prob>_<type>.txt
    ├── main.cpp                     # Main experimental driver
    ├── DynamicNBSim.c++
    ├── FastNBSim.c++
    ├── CreditsAlgorithm.cpp
    └── ... (other headers/source files)
```
    
Requirements:
A C++ compiler with C++11 support (or higher) to utilize libraries like <chrono>, <random>, and <unordered_map>.

Compilation:
It is highly recommended to compile with the -O3 optimization flag, as the code measures exact execution times for benchmarking.
Bash
g++ -O3 main.cpp -o run_experiments

💻 Usage
The program is executed via the command line and now requires exactly 10 arguments (in addition to the executable name).
Syntax:
Bash
./run_experiments <phi> <alpha> <k> <fileName> <p> <gamma> <b> <run_dynamic_baseline> <strategy> <type>

Parameters (in exact order):
<phi> (float): ϕ parameter for the credits algorithm.
<alpha> (float): α parameter for the credits algorithm.
<k> (int): Target size/threshold parameter for the quasi-cliques.
<fileName> (string): Name of the dataset (e.g., youtube, without the .txt extension).
<p> (float): Probability parameter used to fetch the correct sequence file. If p = 0, the system will run "Insert-Only" experiments.
<gamma> (float): γ density parameter.
<b> (float): Specific boundary/tuning parameter for the NBSim structure.
<run_dynamic_baseline> (int/bool): Set to 1 (true) to run dynamic baseline experiments, or 0 (false) to skip them.
<strategy> (string): The sequence strategy to use (e.g., standard, mixed).
<type> (string): The graph generation or sequence type (e.g., erdos, power).
  
Execution Example:
Bash
./run_experiments 0.5 0.8 10 my_graph 0.1 0.9 0.6 1 standard erdos

📊 Output Artifacts
All results are automatically serialized to .csv and .txt files in the ../Esperimenti/<fileName>/ tree. Generated files include:
Performance Logs: times_dynamic_*.csv, times_credits_*.csv
Structural Metrics: results_dynamic_*.csv, density_credits_*.csv, Qs_Size_b=*.csv
Internal State: number_of_credits_*.csv, gamma_degree_*.csv

Remark: all artificial datasets must be generated using generateSequences.py!
